  ┌────────────────────────────┬──────────────────────────────────────┐
  │          ファイル          │                 理由                 │
  ├────────────────────────────┼──────────────────────────────────────┤

  ├────────────────────────────┼──────────────────────────────────────┤
  │ pyproject.toml / setup.cfg │ プロジェクトのビルド・lint設定       │
  ├────────────────────────────┼──────────────────────────────────────┤
  │ requirements.txt           │ 依存パッケージ（再現性のため必須）   │
  ├────────────────────────────┼──────────────────────────────────────┤
  │ .github/workflows/         │ CI/CD パイプライン                   │
  └────────────────────────────┴──────────────────────────────────────┘

  このメモリシステムの概要

  - ~/.claude/projects/[プロジェクト名]/memory/
  にプロジェクトごとのメモリが保存される
  - MEMORY.md がインデックスで、各メモリは個別の .md ファイルになる
  - 会話が始まるたびに MEMORY.md
  が自動的に読み込まれるから、前回の文脈を引き継げる

  他のワークスペースでも使える？

  使えるわ！ただし条件があって、私（Claude
  Code）がそのメモリシステムを使うよう設定されてる必要があるの。

  今回は ~/.claude/CLAUDE.md（グローバル設定）にメモリシステムのルールが書いてあ
  るから動いてるけど、別のワークスペースでも同じグローバル設定を使ってる限り、「
  記憶して」って言えばファイルに残してくれるわよ。

  保存できる記憶の種類

  ┌───────────┬──────────────────────────────────────────────┐
  │  タイプ   │                     内容                     │
  ├───────────┼──────────────────────────────────────────────┤
  │ user      │ あなたの役割・好み・知識レベル               │
  ├───────────┼──────────────────────────────────────────────┤
  │ feedback  │ 「こうしてほしい」「これはやめて」などの指示 │
  ├───────────┼──────────────────────────────────────────────┤
  │ project   │ プロジェクトの背景・目標・締め切りなど       │
  ├───────────┼──────────────────────────────────────────────┤
  │ reference │ 外部リソースの場所（Linear, Slack等）        │
  └───────────┴──────────────────────────────────────────────┘

  ただしコードのパターンやアーキテクチャは記録しない方針ね。それはコード自体を読
  めばわかるから。
