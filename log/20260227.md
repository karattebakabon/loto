# 2026-02-27 作業ログ

## 概要

遺伝的アルゴリズム（GA）によるロト予測機能を実装した。適応度関数（5軸複合スコア）、遺伝的オペレータ（トーナメント選択・順序交叉・突然変異）、GAエンジン、分析レポート、Plotly可視化、CLIの計6ファイルを新規作成し、ユニットテスト（40件）も整備した。これにより、計画していた3つのアルゴリズムが全て実装完了した。

## 実施内容

### 1. 適応度関数（`src/genetic/fitness.py`）

- 5つの評価軸を重み付き合計する複合スコアリング方式を採用:

| # | 評価軸 | 説明 | デフォルト重み |
|---|--------|------|---------------|
| 1 | 合計値スコア | 過去の平均合計値に近いほど高評価（ガウス関数） | 0.25 |
| 2 | 奇偶バランス | 奇数・偶数の比率が 50:50 に近いほど高評価 | 0.20 |
| 3 | 高低バランス | 上半分・下半分の比率が均等なほど高評価 | 0.20 |
| 4 | 連続数字ペナルティ | 連続する数字が少ないほど高評価 | 0.15 |
| 5 | 出現頻度スコア | 過去の頻出数字を多く含むほど高評価 | 0.20 |

- `FitnessConfig` データクラスで重みを管理
- `HistoricalStats` データクラスで過去データの統計値（平均合計値・標準偏差・数字別頻度）を事前計算
- `calculate_fitness_detail()` — 各軸の個別スコアも取得可能（可視化用）

### 2. 遺伝的オペレータ（`src/genetic/operators.py`）

- **初期個体生成** — `random.sample()` で重複なしのランダム数字セットを生成
- **トーナメント選択** — k個体からランダムに選び最良を返す
- **順序交叉（OX: Order Crossover）** — 親1の部分列を保持し、親2の残りで埋める。重複回避に強い手法
- **突然変異** — 1数字をランダムに未使用数字に置換
- 全オペレータがゲーム制約（数字数・範囲・重複なし・ソート済み）を保証

### 3. GAエンジン（`src/genetic/engine.py`）

- `GAConfig` データクラスで全パラメータを管理:
  - 個体群サイズ(200)、世代数(500)、交叉率(0.8)、突然変異率(0.1)、エリート保存(10)、トーナメントサイズ(5)
- `GAResult` データクラスで実行結果を格納（最良個体・適応度推移・最終個体群）
- エリート保存により最良解の消失を防止（単調非減少保証）
- 重複排除済みのユニークな予測番号セットを出力

### 4. 分析レポート（`src/genetic/analyzer.py`）

- 進化レポート: GA パラメータ、世代ごとの適応度推移（最良/平均/最悪）、最良個体の適応度内訳（棒グラフ風）、改善率
- 予測レポート: 予測番号一覧表示

### 5. Plotly 可視化（`src/genetic/visualizer.py`）

- ダークテーマ（グラスモーフィズムデザイン、モンテカルロ・クラスタリングと統一感）
- 4つのセクション:
  1. **適応度推移グラフ** — 最良(シアン)/平均(オレンジ)/最悪(赤)の折れ線
  2. **レーダーチャート** — 最良個体の5軸スコアをペンタゴン表示
  3. **数字分布バーチャート** — 最終世代の全個体で各数字の出現頻度
  4. **予測番号テーブル** — エリート個体をHTMLテーブル表示
- パラメータサマリーカード（個体群・世代・交叉率・突然変異率・エリート・最終適応度）

### 6. CLIエントリーポイント（`src/genetic/__main__.py`）

- `python -m src.genetic` で実行可能
- `argparse` によるオプション:
  - `--game`: loto6 / loto7 / miniloto
  - `--population`: 個体群サイズ
  - `--generations`: 世代数
  - `--crossover-rate`: 交叉率
  - `--mutation-rate`: 突然変異率
  - `--elite`: エリート保存数
  - `--tournament`: トーナメントサイズ
  - `--recent`: 直近N回のみ使用
  - `--predictions`: 予測セット数
  - `--visualize`: HTML可視化
  - `--output-dir`: 出力先

### 7. テストの作成（`tests/test_genetic.py`）

- **TestFitness** — 適応度関数テスト（5件）
  - 戻り型、スコア範囲(0〜1)、全キー存在、各軸範囲、バランス比較
- **TestScoreFunctions** — 各評価軸の個別テスト（6件）
  - 奇偶完全一致、奇偶偏り、高低バランス、連続なし、全連続、頻度範囲
- **TestHistoricalStats** — 統計値計算テスト（6件）
  - 戻り型、平均正値、標準偏差正値、全数字網羅、直近絞り込み、空データ
- **TestOperators** — 遺伝的オペレータテスト（10件）
  - 制約検証(3ゲーム)、ソート済み、初期個体群サイズ、選択動作、選択バイアス、交叉制約(2ゲーム)、突然変異制約、突然変異率0、突然変異率1
- **TestGeneticEngine** — エンジンテスト（13件）
  - 戻り型、予測数、予測制約検証、適応度改善、履歴長、不正キー、コールバック、3ゲーム全対応、ユニーク予測

## 設計判断

### DEAP 不使用の理由
- ロト予測の「離散的・非重複」制約は DEAP の汎用フレームワークだとかえって煩雑
- 他2モジュールと同様、依存を最小限に保つ方が保守しやすい
- 教育的価値（アルゴリズムの中身を理解しやすい）

## CLI使用方法

```powershell
# ロト6（デフォルト）
python -m src.genetic

# ロト7、世代1000、個体群300
python -m src.genetic --game loto7 --generations 1000 --population 300

# ミニロト、可視化付き
python -m src.genetic --game miniloto --visualize

# 予測10セット、直近200回データ
python -m src.genetic --predictions 10 --recent 200

# 全パラメータ指定
python -m src.genetic --game loto7 --population 300 --generations 1000 --crossover-rate 0.85 --mutation-rate 0.15 --elite 20 --tournament 7 --visualize

# ヘルプ
python -m src.genetic --help
```
